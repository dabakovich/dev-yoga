# DevYoga mobile

React Native + Expo SDK 56 (Expo Router, TypeScript). Two native tabs: the
**Tasks** board (list / detail / create-as-sheet) and the **AI Chat**.
iOS-first — Web and Android are not supported in this MVP.

This app runs as a [development build](https://docs.expo.dev/develop/development-builds/introduction/)
(via `expo-dev-client`), **not** Expo Go — it uses native modules (`@expo/ui`,
`expo-glass-effect`) that Expo Go can't load. Builds are done **locally** with
`npx expo run:*`; no paid Apple Developer account is needed.

## Get started

1. Configure the backend URL

   ```bash
   cp .env.example .env
   ```

   `EXPO_PUBLIC_API_BASE` defaults to `http://localhost:3000` — fine for the
   iOS Simulator. On a **real iPhone**, set it to the Mac's LAN IP (find it
   with `ipconfig getifaddr en0`) and make sure both devices are on the same
   Wi-Fi. `EXPO_PUBLIC_*` vars are inlined at bundle time — restart Metro
   after changing (`npx expo start -c`).

2. Install dependencies

   ```bash
   npm install
   ```

3. Build & install the dev build once per platform

   ```bash
   npm run ios       # expo run:ios — builds & installs on the booted iOS Simulator
   ```

   - **Physical iPhone** (free Apple ID, no paid program): `npx expo run:ios --device` and pick
     your device, or open `ios/*.xcworkspace` in Xcode and set *Signing & Capabilities → Team* to
     your personal Apple ID. ⚠️ Free signing **expires after 7 days** — rebuild/reinstall weekly.
     Trust the cert under Settings → General → VPN & Device Management.
   - The native `ios/` and `android/` folders are generated by [CNG](https://docs.expo.dev/workflow/continuous-native-generation/)
     (prebuild) on first run and are gitignored.

4. Start the dev server (after the build is installed)

   ```bash
   npm run dev       # expo start --dev-client (alias: npm start)
   ```

   Open the installed **DevYoga** dev build (not Expo Go); it connects to Metro automatically over
   LAN, or via QR code / the `devyoga://` deep link.

## Structure

File-based routing under [src/app](src/app):

- `(tabs)/index/` — Tasks tab: list (native header filter + sort menus, FAB),
  `[id]` detail; `new.tsx` opens as a form sheet.
- `(tabs)/chat/` — AI chat: persisted transcript, clear-chat header action,
  long-press copy on bubbles.

State: Redux Toolkit + RTK Query ([src/store](src/store)) — `tasks-api` /
`chat-api` query slices with cross-slice tag invalidation (an agent tool call
that mutates tasks refreshes the board automatically), `filters-slice` and
`chat-slice` persisted via redux-persist on MMKV. The chat conversation lives
**only on the device** — the backend is stateless and receives the full
transcript each call.

Native UI comes from `@expo/ui/swift-ui` (header menus, buttons, context
menus) and `expo-glass-effect`; API types mirror the backend DTOs in
[src/utils/api.ts](src/utils/api.ts).
