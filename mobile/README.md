# Welcome to your Expo app 👋

This is an [Expo](https://expo.dev) project created with [`create-expo-app`](https://www.npmjs.com/package/create-expo-app).

## Get started

This app runs as a [development build](https://docs.expo.dev/develop/development-builds/introduction/)
(via `expo-dev-client`), **not** Expo Go — it uses native modules (`@expo/ui`, `expo-glass-effect`)
that Expo Go can't load. Builds are done **locally** with `npx expo run:*`; no paid Apple
Developer account is needed.

1. Install dependencies

   ```bash
   npm install
   ```

2. Build & install the dev build once per platform

   ```bash
   npm run ios       # expo run:ios — builds & installs on the booted iOS Simulator
   npm run android   # expo run:android — builds & installs on an emulator/attached device
   ```

   - **Physical iPhone** (free Apple ID, no paid program): `npx expo run:ios --device` and pick
     your device, or open `ios/*.xcworkspace` in Xcode and set *Signing & Capabilities → Team* to
     your personal Apple ID. ⚠️ Free signing **expires after 7 days** — rebuild/reinstall weekly.
     Trust the cert under Settings → General → VPN & Device Management.
   - The native `ios/` and `android/` folders are generated by [CNG](https://docs.expo.dev/workflow/continuous-native-generation/)
     (prebuild) on first run and are gitignored.

3. Start the dev server (after the build is installed)

   ```bash
   npm start         # expo start --dev-client
   ```

   Open the installed **DevYoga** dev build (not Expo Go); it connects to Metro automatically over
   LAN, or via QR code / the `devyoga://` deep link.

You can start developing by editing the files inside the **src/app** directory. This project uses [file-based routing](https://docs.expo.dev/router/introduction).

## Get a fresh project

When you're ready, run:

```bash
npm run reset-project
```

This command will move the starter code to the **app-example** directory and create a blank **app** directory where you can start developing.
